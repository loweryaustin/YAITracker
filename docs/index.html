Why YAITracker?

Traditional issue trackers were built for a world where humans write all the code. AI coding tools like Cursor, Copilot, and Claude have fundamentally changed development velocity, but project management hasn't caught up.

The most effective developers in the AI age aren't just using one agent — they're orchestrating multiple agents in parallel, sometimes across multiple projects at once. The right tooling makes that difference easier to see, measure, and act on.

Automate the busywork

Keeping issues updated with good notes, accurate time logs, and useful documentation was always valuable but painfully time-consuming. With MCP integration, your AI assistant handles that overhead at the speed of the conversation — more consistent, more detailed tracking with almost zero manual effort.

Understand your full output

See how much work was done overall and how much of it was a result of your effort. When you're orchestrating multiple agents across multiple projects, this is how you measure and improve your real productivity.

AI as a first-class participant

A full Model Context Protocol (MCP) server lets AI assistants in Cursor, Claude Desktop, or any MCP-compatible client create issues, start timers, move cards on the board, and log their own work — all without leaving the editor.

Turn data into better decisions

Track bugs, time, and issues across projects and tag them by framework, language, or workflow. Use velocity analytics, cross-project comparison, and prediction tools to understand what works and estimate more accurately as your team evolves.

Time Tracking

See where the time actually goes

Real-time start/stop timers for both human and AI work. Clock in and out of work sessions, see your daily totals split by human vs. agent time, and review session history — all in one place.

  • Human clock-in/out via the web UI
  • Agent timers managed automatically via MCP
  • Daily summary with human/agent split
  • Weekly timesheet with per-issue breakdown
Issue Tracking

Everything about an issue, in one place

Full issue detail with description, time tracking breakdown (human vs. agent), budget progress bar, activity log, and comments — including AI-generated summaries from completed work.

  • Types, priorities, labels, story points, estimates
  • Time budget bar — logged vs. estimated
  • Human/agent time attribution per entry
  • AI completion summaries logged as comments
Kanban Board

Drag, drop, and stay in flow

A clean drag-and-drop board with real-time updates via htmx. Issues move through backlog, todo, in progress, in review, and done — either manually in the UI or automatically when your AI agent calls begin_work and complete_work.

First-Class MCP Integration

Your AI assistant becomes a full participant in project management. Two commands cover the most common workflow.

// AI agent starts working on an issue
→
begin_work YAIT-42
Starts agent timer, moves issue to in_progress
... agent writes code, runs tests ...
→
complete_work YAIT-42
Stops timer, adds summary comment, moves to done
Completed YAIT-42: Add user settings page
Time logged: 4.2 minutes

Get Started

From Source

git clone https://github.com/
loweryaustin/YAITracker.git
cd YAITracker
make build
# Set a secret (32+ chars)
export YAITRACKER_SECRET="..."
./yaitracker serve
# → http://localhost:8080

Docker

make docker
# Create a .env file
echo 'YAITRACKER_SECRET=...' \
> .env
docker compose up -d
# → http://localhost:8080
