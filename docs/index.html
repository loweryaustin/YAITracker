Why YAITracker?

Traditional issue trackers were built for a world where humans write all the code. AI coding tools like Cursor, Copilot, and Claude have fundamentally changed development velocity, but project management hasn't caught up.

Automate the busywork

Keeping issues updated with good notes, accurate time logs, and useful documentation was always valuable but painfully time-consuming. With MCP integration, your AI assistant handles that overhead at the speed of the conversation — so you get more consistent, more detailed tracking with almost zero manual effort.

Understand your full output

See how much work was done overall and how much of it was a result of your effort. When you're orchestrating multiple agents across multiple projects, this is how you measure and improve your real productivity.

AI as a first-class participant

A full Model Context Protocol (MCP) server lets AI assistants in Cursor, Claude Desktop, or any MCP-compatible client create issues, start timers, move cards on the board, and log their own work — all without leaving the editor.

Turn data into better decisions

Track bugs, time, and issues across projects and tag them by framework, language, or workflow. Use velocity analytics, cross-project comparison, and prediction tools to understand what works and estimate more accurately as your team evolves.

Features

Issue Tracking

Jira-style issues with types, priorities, labels, epics, and parent/child hierarchy.

Kanban Board

Drag-and-drop board with real-time updates via htmx.

Time Tracking

Real-time start/stop timers, work sessions, human vs. agent time split.

Velocity Analytics

Sprint velocity, cycle time, estimation accuracy, and project health.

Cross-Project Comparison

Compare metrics across projects by technology tags.

Project Prediction

Estimate timelines for new projects based on historical data.

REST API

JSON API with OAuth2 authentication for mobile apps and integrations.

Web UI

Server-rendered HTML with htmx and Alpine.js. No SPA framework needed.

Simple Deployment

Single binary with embedded assets. SQLite for zero-config use today; more backends planned.

First-Class MCP Integration

Your AI assistant becomes a full participant in project management. Connect any MCP-compatible client and let it manage issues, track time, and query analytics.

Workflow Tools

Two commands cover the most common workflow:

→ begin_work YAIT-42
Starts timer, moves to in-progress
... agent does the work ...
→ complete_work YAIT-42
Stops timer, adds summary, moves to done

Get Started

From Source

git clone https://github.com/
loweryaustin/YAITracker.git
cd YAITracker
make build
# Set a secret (32+ chars)
export YAITRACKER_SECRET="..."
./yaitracker serve
# → http://localhost:8080

Docker

make docker
# Create a .env file
echo 'YAITRACKER_SECRET=...' \
> .env
docker compose up -d
# → http://localhost:8080
Full documentation on GitHub →

Architecture

yaitracker serve
→ chi Router → HTML handlers (htmx/Alpine.js)
→ JSON API (/api/v1, OAuth2)
→ Static assets (embedded)
yaitracker mcp
→ MCP Server (stdio or HTTP) → tools + resources

Both commands share the same SQLite database. The entire application compiles to a single binary with all assets embedded.
